Основное понятие в теории БВ - модель данных.
Модель данных характеризует способ организации данных и основные методы
доступа к ним.

Пользователи:
1 - прикладные программисты
2 - конечные пользователи
3 - админы БД

Пример процессора языка запросов - SQL

Система БД разедляется в основном на:
- сервер
- набор клиентов (внешних интерфейсов)

Сервер - СУБД
Клиенты - приложения

В реляционных БД данные воспринимаются пользователем как таблицы.
Особенность - способность поддерживать свзи между элементаи данных.

Таблицы состоят из строк и столбцов
Таблицы называются отношениями (relations)
Строки таблицы - кортежи (tuples)
Колонки таблицы - атрибуты
Количество кортежей - кардинальное число
Количество атрибутов - степень отношения

Для идентификации строк в таблицах и для связи таблиц между собой используются
ключи.
Потенциальный ключ - комбиация атрибутов таблицы
Ключ может состоять толко из одного атрибута таблицы
Ключи нужны для адресации строк (записей).

В SQL выделяют:
DDL - Data Definition Language - группа операторов определения данных
DML - Data Manipulation Language - группа операторов манипулирования данными
LCL - Data Control Language - группа операторов управления привилегиями доступа
